# 竞品分析 Prompt

## 系统概述

我正在设计一个名为 **Mindflow** 的智能体操作系统（Agent OS），请帮我分析当前市场上是否存在类似的竞品。

## Mindflow 核心定位

**定位**: 运行在 Windows/Linux/macOS 之上的应用层智能体操作系统

**核心特点**:
1. **三层架构**
   - 应用层: Extension 插件生态（知识库、生活记录、计划管理等）
   - 服务层: Extension Manager、Agent Orchestrator、Tool Registry
   - 底层能力: 语音I/O、视觉理解、文件系统、网络、系统控制

2. **核心功能**
   - Extension 插件系统（动态加载、生命周期管理）
   - Tool Registry 工具注册表（统一的能力调用接口）
   - 多 LLM Provider 支持（Claude、GPT、DeepSeek 等可切换）
   - Agent 编排和调度（Subagent 并行执行）
   - Context 管理（长对话压缩、重要信息提取）
   - Session 管理（会话持久化和恢复）
   - Schedule 定时任务

3. **底层能力封装**
   - 语音输入输出（Whisper + TTS）
   - 视觉理解（GPT-4V/Claude Vision）
   - 文件系统操作（封装 os.*）
   - 网络请求（封装 requests）
   - 系统控制（封装 subprocess）
   - 数据持久化（SQLAlchemy）

4. **应用场景**
   - 个人知识库管理
   - 生活事件记录和分析
   - 计划管理和智能推动
   - 每日复盘和反思
   - 用户画像和个性化
   - [未来: 更多生活场景应用]

5. **技术栈**
   - 语言: Python
   - UI: Gradio / 桌面应用（Tauri/Electron）
   - AI: LangGraph + 多 LLM Provider
   - 数据库: SQLite/PostgreSQL
   - 向量数据库: Chroma/Weaviate

## 分析要求

请从以下维度分析竞品：

### 1. 直接竞品
- 定位为"智能体操作系统"或"Agent OS"的产品
- 提供 Extension/插件生态的 AI 助手
- 支持多 Agent 编排的系统

### 2. 间接竞品
- 个人 AI 助手（如 Notion AI、Mem.ai）
- 开发者 AI 助手（如 Goose、Cursor、GitHub Copilot）
- 知识库管理工具（如 Obsidian、Logseq）
- 生活记录工具（如 Day One、Journey）

### 3. 分析维度
对每个竞品，请分析：
- **产品名称和定位**
- **核心功能**（列出 3-5 个关键功能）
- **架构特点**（是否有插件系统、Agent 编排等）
- **技术栈**（编程语言、框架）
- **目标用户**
- **商业模式**（开源/闭源、免费/付费）
- **与 Mindflow 的相似度**（1-10 分，10 分最相似）
- **差异化优势**（Mindflow 相比该竞品的优势）
- **可借鉴之处**（该竞品值得学习的地方）

### 4. 市场空白分析
- 当前市场上缺少什么？
- Mindflow 的独特价值是什么？
- 哪些功能组合是市场上没有的？

### 5. 建议
- 是否建议继续开发 Mindflow？
- 如果是，应该重点突出哪些差异化特性？
- 如果否，原因是什么？
