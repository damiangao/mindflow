```bash
cat planning/v1.0/PLAN.md

```

### 了解版本间的变化
```bash
cat planning/v1.0.1/CHANGES.md
diff planning/v1.0/PLAN.md planning/v1.0.1/PLAN.md
```

### 按版本时间线规划
- 预期完成时间

- 关键依赖关系

### 追踪项目进度
- 比较计划 vs 实际

- 调整后续计划

## 版本标签 (Git Tags)

```bash
# 创建v1.0 tag
```

## 重要提示

### 关于版本号
- **v1.0.x**: bug修复或小的优化
- **v1.x**: 新功能或重大改进
- **vX.0**: 新阶段或大的架构变化

### 关于发布时间
- 上面的日期是**预估**，会根据实际进度调整
- 每周检查一次是否需要调整时间线
- 优先级是完成质量而非赶时间

### 关于向后兼容
- v1.x系列内部兼容
- 升级到v2.0时可能需要数据迁移
- 升级到v3.0时需要重新配置认证
